Flask + Redis

HTML Pages for the Visit Counter App

This project uses two HTML templates that work together to create a simple visit tracking app.

🏠
home.html — the landing page

This is the first page the user sees when visiting the app at http://localhost:5001. It has no dynamic content — it is pure static HTML. Its only job is to welcome the user and provide a "View visit count" button that links to /count.

When the button is clicked, the browser sends a GET request to the /count route in Flask.

📊
visits.html — the counter page

When Flask receives a request on /count, it calls r.incr('visits') — this tells Redis to increment the visits key by 1 and return the new value. Flask then passes that number to the template: 

return render_template('visits.html', count=visits)
💡
Why two separate pages?
Splitting the welcome page and the counter page keeps responsibilities clean. home.html is static and loads instantly with no Redis dependency. visits.html is dynamic and only hits Redis when actually needed. If Redis is down, only the /count route breaks — the home page still loads fine.
🔄
The full request flow
User clicks the View visit count button
Browser sends GET /count to Flask
Flask calls r.incr('visits') → Redis increments the key
Flask renders visits.html with the new count value
User sees the updated counter in the browser
